213. 打家劫舍 II

题目描述

你是一个专业的小偷，计划偷窃沿街的房屋，每间房内都藏有一定的现金。这个地方所有的房屋都 围成一圈 ，这意味着第一个房屋和最后一个房屋是紧挨着的。同时，相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警 。

给定一个代表每个房屋存放金额的非负整数数组，计算你 在不触动警报装置的情况下 ，今晚能够偷窃到的最高金额。

示例 3：

提示：

  • 1 <= nums.length <= 100
  • 0 <= nums[i] <= 1000
题解

所有房子都围成一圈，也就是说是一个环形数组。因为不能偷相邻的两间房屋，所以要是偷取第一个，最后一个就无法偷取；要是偷取最后一个，第一个就无法偷取。那么怎么才能做到这个限制呢？只要截取数组[0,n-1]，[1,n]就可以了，这样就可以标志偷取范围是第一间到倒数第二间；第二间到倒数第一间。相应的我们也无需再考虑首尾相连的问题了。然后这两次的最大值就是能偷取到的最大金额。

再来想一下状态转移方程。设dp[i]为在i号位置能获取到的最大金额。如果不偷取第i号位置，那么dp[i]=dp[i-1]；如果偷取第i号位置，那么dp[i] = dp[i-2] + nums[i]。这样就能得到方程：dp[i] = Math.max(dp[i-1], dp[i-2] + nums[i]);

1388. 3n 块披萨

看完打家劫舍Ⅱ，再来看一下3n块披萨。

题目描述

给你一个披萨，它由 3n 块不同大小的部分组成，现在你和你的朋友们需要按照如下规则来分披萨：

  • 你挑选 任意 一块披萨。
  • Alice 将会挑选你所选择的披萨逆时针方向的下一块披萨。
  • Bob 将会挑选你所选择的披萨顺时针方向的下一块披萨。
  • 重复上述过程直到没有披萨剩下。

每一块披萨的大小按顺时针方向由循环数组 slices 表示。

请你返回你可以获得的披萨大小总和的最大值。

示例 2：

提示：

  • 1 <= slices.length <= 500
  • slices.length % 3 == 0
  • 1 <= slices[i] <= 1000
题解

其实不用关心有两个朋友也要获取披萨，只要关心选择一个位置，那么周围两个都不能选择就行了，也就是说不能选择相邻的披萨。

这个题跟打家劫舍Ⅱ很相似。同样的都是一个环形数组，不允许连续挑选，我们把数组分成[0,n-1],[1,n]两个来解决首位相连的问题。

跟打家劫舍不同的是，在这道题中不需要从第一块披萨开始选，也就是说每次选取的下标可以是随机的，有跨度的，只要满足不相邻就行。

再来看一下状态转移方程：设dp[i][j]为在前i个元素中选择了j个元素的最大和。如果不选择第i个元素，那么dp[i][j] = dp[i-1][j]；如果选择第i个元素，那么dp[i][j] = dp[i-2][j-1] + nums[i]。

这样就能得到方程：dp[i][j] = Math.max(dp[i-2][j-1] + nums[i], dp[i-1][j])

那i和j的取值范围又是多少呢？i是所有的元素个数，即slices.length。j是能选择的个数，也就是slices.length/3
